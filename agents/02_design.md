## デザイン原則

### 絶対禁止（AIっぽいデザイン回避）
```
❌ 紫〜青のグラデーション背景
❌ 絵文字の装飾的使用
❌ Inter, Roboto, Arial フォント
❌ 均一な角丸カード群
❌ 無個性なアイコンセット
❌ 白背景＋青アクセントの組み合わせ
❌ 過度に整列された完璧なグリッド
❌ 汎用的なイラスト素材
```

### 推奨アプローチ
```
✓ カフェの実際の素材感を反映
✓ 木目、紙、リネンなどのテクスチャ
✓ 手書き風のアクセント要素
✓ 和モダンの繊細さ
✓ 印刷したくなる美しさ
✓ 情報の優先度が明確な階層構造
```

```css

  font-family: "Shippori Mincho", "Noto Serif JP", serif;
```

```css

  --text-xs: 0.75rem;    /* 12px - 注釈 */
```

## 絶対遵守事項：画像生成連携
```
【必須】デザインに画像が必要な場合、Gemini 2.5 Flash Image APIを使用して生成すること
【必須】画像生成スクリプト generate_ng_images.py を参考に、用途に応じたスクリプトを作成
【必須】生成した画像は images/ フォルダ配下に適切なサブフォルダを作成して保存
【必須】data.js、script.js、styles.css を一貫して更新すること
```

### 画像生成フロー
```
1. デザインで画像が必要な箇所を特定
2. 画像の用途・サイズ・縦横比を決定
3. Pythonスクリプトでプロンプトを作成
4. Gemini 2.5 Flash Image API で画像生成
5. images/ フォルダに保存
6. data.js に画像パスとalt属性を追加
7. script.js で画像表示ロジックを実装
8. styles.css で画像カードスタイルを追加
```
