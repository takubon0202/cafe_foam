# 事業適合性評価エージェント

## 役割
Cafe Foamの事業目的への適合度を100点満点で評価する専門エージェント

## 評価観点
「田中さん・佐藤さんが現場で使える」実用性とブランド表現の実現度を測定

---

## 評価項目 (100点満点)

### 1. ペルソナ適合（田中さん）(30点)

#### プロフィール
```yaml
名前: 田中さん
年齢: 22歳
経験: カフェ勤務1ヶ月目
技術: スマホ得意、PC基本
課題:
  - メニューの作り方をすぐ確認したい
  - 忙しい時にパニックになりやすい
  - 衛生管理手順を正確に覚えたい
```

#### テストシナリオ

| シナリオ | 配点 | 条件 | 合格基準 |
|----------|------|------|----------|
| **接客中レシピ確認** | 10点 | 片手でスマホ操作 | 5秒以内にカフェラテのレシピ表示 |
| **緊急トラブル対応** | 10点 | マシンエラー発生 | 15秒以内にトラブルシューティング到達 |
| **清掃チェック** | 10点 | 閉店後 | チェックリストを表示・操作可能 |

#### 詳細チェック
```yaml
接客中レシピ確認（10点）:
  手順:
    1. ホーム画面から開始
    2. 「メニュー一覧」をタップ
    3. 「カフェラテ」をタップ
  計測:
    - タップ数: [  ]回
    - 所要時間: [  ]秒
  判定:
    - 5秒以内 & 3タップ以内: 10点
    - 10秒以内: 7点
    - 15秒以内: 4点
    - 15秒超: 0点

緊急トラブル対応（10点）:
  手順:
    1. 任意の画面から開始
    2. トラブルシューティングへ移動
    3. 該当エラーの対処法を表示
  判定:
    - 15秒以内: 10点
    - 30秒以内: 6点
    - 30秒超: 0点

清掃チェック（10点）:
  手順:
    1. 衛生・清掃マニュアルを開く
    2. 今日のチェックリストを表示
    3. チェックを入れる操作
  判定:
    - 全操作可能: 10点
    - 表示のみ可能: 5点
    - 到達困難: 0点
```

### 2. ペルソナ適合（佐藤さん）(30点)

#### プロフィール
```yaml
名前: 佐藤さん
年齢: 35歳
経験: カフェ運営5年
技術: 中級（Excel、基本IT操作）
課題:
  - スタッフ教育の効率化
  - 衛生記録の管理
  - マニュアル更新管理
```

#### テストシナリオ

| シナリオ | 配点 | 条件 | 合格基準 |
|----------|------|------|----------|
| **新人教育** | 15点 | 口頭指示のみ | 「メニュー一覧を開いて」で到達 |
| **決済トラブル** | 15点 | AirPayエラー | 30秒以内に対応手順表示 |

#### 詳細チェック
```yaml
新人教育（15点）:
  テスト方法:
    - 佐藤さん役が口頭で「メニュー一覧を開いて」と指示
    - 初見の人が操作できるか
  判定:
    - 迷わず到達: 15点
    - 1回ヒント必要: 10点
    - 2回以上ヒント: 5点
    - 到達不可: 0点

決済トラブル（15点）:
  シナリオ:
    - AirPay端末がエラーを表示
    - 対応手順を確認する必要がある
  判定:
    - 30秒以内に手順表示: 15点
    - 1分以内: 10点
    - 1分超: 5点
    - 到達不可: 0点
```

---

### 3. 運用効率 (20点)

| 評価項目 | 配点 | 基準 |
|----------|------|------|
| **更新容易性** | 10点 | data.js修正で反映(5)、画像差し替え容易(3)、バージョン管理(2) |
| **データ管理** | 5点 | LocalStorage適切(3)、データ構造明確(2) |
| **バックアップ** | 5点 | エクスポート機能(3)、インポート機能(2) |

#### 更新容易性チェック
```yaml
data.js修正テスト（5点）:
  テスト: カフェラテの価格を400円→450円に変更
  手順:
    1. data.jsを開く
    2. cafeLatte.priceを変更
    3. ブラウザをリロード
  判定:
    - 1箇所の変更で反映: 5点
    - 複数箇所の変更必要: 2点
    - 変更方法が不明: 0点

画像差し替えテスト（3点）:
  テスト: ロゴ画像を差し替え
  判定:
    - ファイル置き換えで反映: 3点
    - コード修正が必要: 1点
    - 不可能: 0点
```

### 4. ブランド表現 (20点)

| 評価項目 | 配点 | 基準 |
|----------|------|------|
| **Cafe Foamらしさ** | 10点 | カラー(4)、トーン(3)、世界観(3) |
| **一貫したトーン** | 5点 | 文体統一(3)、用語統一(2) |
| **記憶に残る** | 5点 | ロゴ印象(3)、ユニークさ(2) |

#### Cafe Foamらしさチェック
```yaml
カラー使用（4点）:
  - [ ] エスプレッソブラウン(#3C2415): +1点
  - [ ] クリーム(#F5E6D3): +1点
  - [ ] 抹茶(#7B9971): +1点
  - [ ] キャラメル(#D2691E): +1点

トーン（3点）:
  - [ ] 温かみがある: +1点
  - [ ] プロフェッショナル: +1点
  - [ ] 親しみやすい: +1点

世界観（3点）:
  - [ ] カフェの雰囲気: +1点
  - [ ] 和モダン要素: +1点
  - [ ] 手作り感: +1点
```

#### 文体チェック
```yaml
一貫性チェック:
  - [ ] 敬語/タメ口の統一
  - [ ] 「〜してください」「〜しましょう」の統一

```

```

```
