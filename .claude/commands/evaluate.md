# 品質評価システム実行

4つの専門評価エージェントが並行して評価を行い、総合判定します。

## 評価アーキテクチャ

```
┌─────────────────────────────────────────────────────────┐
│                  総合評価エージェント                      │
│              agents/07_evaluator.md                     │
├─────────────────────────────────────────────────────────┤
│  ┌──────────────┐ ┌──────────────┐                     │
│  │07a_design    │ │07b_ux        │                     │
│  │デザイン評価   │ │UX評価        │                     │
│  │100点満点    │ │100点満点    │                     │
│  └──────────────┘ └──────────────┘                     │
│  ┌──────────────┐ ┌──────────────┐                     │
│  │07c_tech      │ │07d_business  │                     │
│  │技術評価      │ │事業適合性評価 │                     │
│  │100点満点    │ │100点満点    │                     │
│  └──────────────┘ └──────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

## 実行手順

### Step 1: 専門エージェント読み込み
```yaml
読み込みファイル:
  - agents/07_evaluator.md (総合評価)
  - agents/07a_design_evaluator.md (デザイン)
  - agents/07b_ux_evaluator.md (UX)
  - agents/07c_tech_evaluator.md (技術)
  - agents/07d_business_evaluator.md (事業適合性)
```

### Step 2: 並行評価実行
4つの専門エージェントが同時に評価を実行

### Step 3: 総合判定
- 4カテゴリの平均スコアを算出
- 合格/条件付き合格/不合格を判定

## 合格基準（厳格化）

### 総合判定
| 条件 | 判定 | アクション |
|------|------|-----------|
| 全カテゴリ80点以上 & 平均80点以上 | **合格** | デプロイ可能 |
| 全カテゴリ70点以上 & 平均75点以上 | **条件付き合格** | 軽微修正後デプロイ |
| 1カテゴリでも70点未満 | **不合格** | 差し戻し |

### 致命的減点（自動不合格）
- デザイン: AIっぽい要素の使用（紫グラデ、絵文字装飾）
- UX: 4タップ以上で目的未達成
- 技術: Lighthouse 60点未満
- 事業適合性: ペルソナテスト全不合格
